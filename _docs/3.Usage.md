The default route for the LogViewer is `http://yourdomain.com/log-viewer`.

## Dashboard

The dashboard provides a quick overview of your log statistics:
- Total logs count.
- Percentage of each log level (Emergency, Alert, Critical, Error, Warning, Notice, Info, Debug).

## Logs List

View and manage all your log files in one place. You can:
- **View**: See the entries for a specific log file.
- **Download**: Download the log file directly.
- **Delete**: Permanently delete a log file.

## Programmatic Usage

You can also interact with LogViewer programmatically using the `LogViewer` facade or by injecting the `Squipix\LaravelLogViewer\Contracts\LogViewer` interface.

### Facade Usage

```php
use Squipix\LaravelLogViewer\Facades\LogViewer;

// Get all logs
$logs = LogViewer::all();

// Get logs count
$count = LogViewer::count();

// Get entries for a specific date
$entries = LogViewer::entries('2026-03-09');

// Download a log file
return LogViewer::download('2026-03-09');
```

### API Usage

If you prefer to build your own UI, you can use the LogViewer API methods:

```php
/** @var  \Squipix\LaravelLogViewer\Contracts\LogViewer  $logViewer */
$logViewer = app(Squipix\LaravelLogViewer\Contracts\LogViewer::class);

// Get stats
$stats = $logViewer->stats();

// Get the log levels list
$levels = $logViewer->levels();
```

## Artisan Commands

LogViewer comes with several Artisan commands to help you manage your logs:

- `php artisan log-viewer:check`: Check if your installation meets the requirements.
- `php artisan log-viewer:publish`: Publish the config and translations files.
- `php artisan log-viewer:clear`: Clear all generated log files.
- `php artisan log-viewer:stats`: Display log statistics in the console.
